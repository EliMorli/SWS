Section 07

Module 4: Change Order Management

Overview

Hartford had 12 approved change orders totaling $268,005.20 - nearly 31% of the original contract. Managing COs manually via signed PDFs creates risk of missed approvals and unbilled work. This module provides a full CO lifecycle from request to approval to billing integration.

Change Order Workflow

  1. Initiate RFQ - Field super or PM identifies extra work, submits Request for Quote in app with scope, photos, estimated cost
  2. Internal Estimate - PM prices the CO using the estimating engine, generates draft CO document
  3. Submit to GC - CO submitted via email with PDF attachment (DocuSign envelope auto-created for GC signature)
  4. GC Review - System tracks status: Submitted / Under Review / Approved / Rejected / Partially Approved
  5. Approval - GC signs via DocuSign. System auto-updates contract total and flags for billing on next pay app
  6. Billing Integration - Approved CO amount automatically appears as a line in the next pay application (G703 continuation sheet)

Module 6: Lien Release Tracking

Overview

California lien law is strict and unforgiving. Missing a lien deadline means losing the right to lien permanently. SWS must manage both its OWN lien releases (provided to GC to receive payment) and CONDITIONAL releases FROM subcontractors and material suppliers before disbursing funds. This module is a legal risk-management tool.

Lien Release Types

Sub Unconditional WaiverSub to SWSAfter SWS payment to sub clears
L&W Supply ConditionalL&W to SWSMonthly, matching their billing to SWS
L&W Supply UnconditionalL&W to SWSMonthly, after joint check clears

All background automation runs via Sidekiq workers. Scheduled jobs run nightly at 6:00 AM PT. All notifications respect user preference settings (email, SMS, in-app).

Automation Workflows

Pay Application Auto-Draft

When a project phase is marked complete: 1) System auto-calculates the billable amount for that phase. 2) Generates a draft pay application pre-populated with prior billing history, CO totals, and retention calculation. 3) Notifies the billing user to review and submit. 4) One-click submission generates PDF and sends to GC email on file.

Lien Release Chaining

When a payment is recorded: 1) System checks if a conditional lien release exists for the invoice. 2) If conditional was sent and payment received, auto-prompts to generate unconditional release. 3) DocuSign envelope created for e-signature. 4) Once signed, unconditional release stored and linked to payment record.

Insurance Renewal Workflow

60 days before any policy expiry: 1) Alert sent to Aaron with policy details. 2) Follow-up at 30 and 15 days. 3) At expiry: System flags all projects using that policy. 4) New invoice generation is blocked until renewed policy uploaded. 5) On upload: All GC contacts auto-notified with updated COI.

Data Security

  • All data encrypted at rest (AWS RDS encryption, S3 SSE-S3)
  • All data encrypted in transit (TLS 1.3 minimum, Cloudflare enforced)
  • S3 documents served via signed URLs only (15-min expiry). No public buckets.
